\subsection{System Concept Definition}

The \gls{marv} system implements a three-subsystem architecture with hierarchical control and distributed sensing. Figure~\ref{fig:system-functional} presents the system functional block diagram showing subsystem responsibilities and data flow. Figure~\ref{fig:system-architecture} presents the system architectural definition showing hardware components and communication interfaces.

The \gls{snc} subsystem executes supervisory control including state machine coordination, \gls{navcon} decision logic, and \gls{scs} protocol management. The \gls{ss} subsystem provides environmental sensing with camera-based line detection, color classification, and angle measurement. The \gls{mdps} subsystem implements motion control with differential drive kinematics, wheel odometry, and maneuver execution. Communication employs \gls{scs} protocol over UART with turn-based packet exchange enabling coordinated multi-subsystem operation.
